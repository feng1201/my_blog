# 📝 如何添加新文章

## 🚀 超简单的添加方式

### 方法1：直接在GitHub网页上创建（推荐）

1. **进入posts文件夹**
   - 在GitHub仓库中点击 `posts` 文件夹

2. **创建新文件**
   - 点击 `Add file` → `Create new file`
   - 文件名格式：`YYYY-MM-DD-文章标题.md`
   - 例如：`2025-10-18-我的第一篇技术博客.md`

3. **添加文章内容**
   ```markdown
   ---
   title: "我的第一篇技术博客"
   date: "2025-10-18"
   tags: ["技术", "学习"]
   author: "你的名字"
   ---

   # 我的第一篇技术博客

   这里写你的文章内容...

   ## 小标题

   - 列表项1
   - 列表项2

   | 表格列1 | 表格列2 |
   |---------|---------|
   | 内容1   | 内容2   |
   ```

4. **提交文件**
   - 滚动到页面底部
   - 填写提交信息：`添加新文章：我的第一篇技术博客`
   - 点击 `Commit new file`

5. **更新文章索引**
   - 点击 `posts.json` 文件
   - 点击编辑按钮（铅笔图标）
   - 在数组中添加新文章信息：
   ```json
   {
     "filename": "2025-10-18-我的第一篇技术博客.md",
     "title": "我的第一篇技术博客",
     "date": "2025-10-18",
     "tags": ["技术", "学习"],
     "author": "你的名字",
     "excerpt": "这里写文章的简短描述..."
   }
   ```

## 📋 文章格式说明

### Front Matter（文章头部信息）
每篇文章开头必须包含以下信息：

```yaml
---
title: "文章标题"          # 必需
date: "YYYY-MM-DD"        # 必需，格式：2025-10-18
tags: ["标签1", "标签2"]   # 可选，用于分类
author: "作者名"          # 可选
excerpt: "文章摘要"       # 可选，用于首页显示
---
```

### Markdown内容
支持完整的Markdown语法：

- **标题**：`# ## ###`
- **粗体**：`**文字**`
- **斜体**：`*文字*`
- **列表**：`- 项目` 或 `1. 项目`
- **链接**：`[文字](URL)`
- **图片**：`![描述](图片URL)`
- **表格**：使用 `|` 分隔
- **代码**：使用 ``` 包围

## 💡 小贴士

1. **文件命名**：使用日期开头便于排序
2. **标签使用**：保持标签简洁，便于分类
3. **图片管理**：将图片上传到 `images` 文件夹
4. **定期备份**：GitHub本身就是很好的备份
5. **预览功能**：GitHub有Markdown预览功能

## 🆘 常见问题

**Q: 文章没有显示在博客上？**
A: 检查是否更新了 `posts.json` 文件

**Q: 图片无法显示？**
A: 确保图片已上传到 `images` 文件夹，并使用正确的相对路径

**Q: 表格格式不对？**
A: 检查表格语法，确保每行的 `|` 数量一致

**Q: 想修改博客样式？**
A: 编辑 `index.html` 中的CSS部分
